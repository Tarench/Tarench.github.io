自然语言处理 基于预训练模型的方法原创

《自然语言处理：基于预训练模型的方法》 ——车万翔 郭江 崔一鸣

# 第一章、绪论

  • NLP：研究用计算机理解和生成自然语言的各种理论和方法
  • 语料库：面向某一自然语言处理任务所标注的数据

# 1.NLP的难点

  1. 抽象性：语言由抽象符号构成，每个符号背后都对应着现实世界或人们头脑中的复杂概念
  2. 组合性：有限的字有很多种组合，成不同含义的句子
  3. 歧义性：一词多义的情况
  4. 进化性：新词汇or旧词汇被赋予新的意义
  5. 非规范性：简写or谐音梗
  6. 主观性
  7. 知识性：根据上下文理解
  8. 难移植性：NLP所设计的任务和领域众多，且之间差异大

# 2.NLP的任务层级

# 3.NLP的任务类别

  1. 回归问题：将输入文本映射为一个连续的数值 eg.作文的打分，预测任务
  2. 分类问题：判断一个输入文本所属的类别
  3. 匹配问题：判断两个输入文本之间的关系、识别两个输入文本之间的相似性
  4. 解析问题：对文本中的词语进行标注或识别词语之间的关系 eg.词性标注、句法分析
  5. 生成问题：根据输入生成一段自然语言 eg.机器翻译、图像描述

# 4.NLP的研究对象与层次

# 第二章、NLP基础

# 1.文本的表示

  • 发展历程：One-hot表示 → 分布式表示 → 词向量
  • 基于机器学习的NLP技术：其最本质的思想是==将文本表示为向量，其中的每一维代表一个特征。在进行决策的时候，只要对这些特征的相应值进行加权求和，就可以得到一个分数用于最终的判断==

一种非常简单的将原始文本转为词向量的方法：令向量x的每一维表示某个词在该文本中出现的次数，然后根据每个词的重要性进行加权

  • 文本项目表示也可用于计算两个文本之间的相似度，即使用余弦函数等度量函数表示两个向量之间的相似度，并应用于信息检索等任务

# PS.关于文本相似度

# 1.1 独热表示(One-hot Encoding)

  • 用一个词表大小的向量表示一个词，然后将词表中的第i个词w~i~表示为向量

在该向量中，词表中第i个词在第i维上被设置为1，其余维度均为0

缺点

  1. 不同词使用完全不同的向量进行表示，导致即使两个词在语义上很相似，但通过余弦函数来度量相似度时值却为0
  2. 在应用于ML的方法时会导致数据稀疏（Data Sparsity)问题

eg.假设在训练数据中只见过“漂亮”，在测试数据中出现了“美丽”，虽然它们之间很相似，但是系统仍然无法恰当地对“美丽”进行加权。由于数据稀疏问题，导致当训练数据规模有限时，很多语言现象没有被充分地学习到

  • 为缓解Data Sparsity的问题，传统做法是除了词自身，再提取更多和词相关的泛化特征，如词性特征、词义特征、词聚类特征等

以语义特征为例，通过引入WordNet等语义词典，可以获知“漂亮”和“美丽”是同义词，然后引入它们的共同语义信息作为新的额外特征，从而缓解同义词的独热表示不同的问题

# 1.2 分布式表示

One-hot Encoding容易导致Data Sparsity，虽然可以通过引入特征的方法缓解，但设计特征比较耗费精力。想一个可以自动提取特征并设置相应特征值的方法

  • 分布式语义假设：词的含义可通过其上下文的分布进项表示
  • 上下文的选择方式有多种 可以使用词在句子中的一个固定窗口内的词作为其上下文，也可以使用所在的文档本身作为上下文。前者得到的词表示将更多地反映词的局部性质：具有相似词法、句法属性的词将会具有相似的向量表示。而后者将更多地反映词代表的主题信息

eg.

  • 创建词语共现次数表，表中每一项表示一个词w~i~与另一个词w~j~（上下文）在同一个句子中共现的次数，每个词与自身的共现次数设为0

  • 表中每一行代表一个词的向量，可以通过向量计算词之间的相似度

使用上下文的共现频次作为词的向量表示存在的问题

  1. 高频词舞蹈结果错误，如标点符号、人称代词等与其他词的共现频次都较高
  2. 共现频次无法反应词之间的高阶关系，如假设词“A”与“B”共现过，“B”与“C”共现过，“C”与“D”共现过，通过共现频次，只能获知“A”与“C”都与“B”共现过，它们之间存在一定的关系，而“A”与“D”这种高阶的关系则无法知晓
  3. 仍存在稀疏性的问题，即向量中仍有很多0

解决方式：点互信息和奇异值

# 点互信息(PMI)

  • 解决高频词误导计算结果的问题
  • 如果一个词与很多词共现，则降低其权重；反之，如果一个词只与个别词共现，则提高其权重


P （w， c）、P （w）、P （c）分别是w与c的共现概率，以及w和c分别出现的概率，可以通过极大似然估计(MLE) 计算

  • 当某个词与上下文之间共现次数较低时，可能会得到负的PMI值，在应用中通常使用Positive PMI：
  • 计算PPMI的代码： 其中定义M为类似上面介绍过的共现矩阵的形式，即

    其中V为词表，C为全部的上下文
import numpy as np

def pmi(M, positive=True)
  col_totals = M.sum(axis=0) # 按列求和
  row_totals = M.sum(axis=1) # 按行求和
  total = col_totals.sum() # 总频次
  expect = np.outer(row_totals, col_totals) / total # 计算每个元素的分子部分
  M = M / expected
  with np.errstate(divide='ignore'): # 不显示log(0)的警告
    M = np.log(M)
  M[np.isinf(M)] = 0.0 # 将log(0)置为0
  if positive:
    M[M < 0] = 0.0
  return M

M_pmi = pmi(M)
np.set_printoptions(precision=2) # 结果保留两位小数
print(M_pmi)

# 奇异值分解(SVD)

  • 解决共现频次无法反应词之间高阶关系的问题

原理分析如下：

对共现矩阵M进行奇异值分解：

其中：、 为正交矩阵，满足 是由r个奇异值(Singular Value)构成的对角矩阵

若在中仅保留个()最大的奇异值，和也只保留相应的维度，则被称为截断奇异值分解(Truncated Singular Value Decomposition)，截断奇异值分家实际上是对矩阵的低秩近似 通过截断奇异值分解所得到的矩阵U中的每一行，则为相应词的d维向量表示，该向量一般具有连续、低维和稠密的性质。由于U的各列相互正交，因此可以认为词表示的每一维表达了该词的一种独立的“潜在语义”，所以这种方法也被称作潜在语义分析（Latent Semantic Analysis，LSA）

  • Python的numpy.linalg库中内置了SVD函数，输入共现矩阵调用函数即可，如：
U, s, Vh = np.linalg.svd(M_pmi)

分布式表示存在的问题：

  1. 共现矩阵较大时，SVD的运行速度慢
  2. 若想在原来语料库的基础上增加更多数据，需要重新运行SVD分解算法，代价搞
  3. 分布式只能用于表示比较短的单元
  4. 一旦训练完成，无法修改，无法根据具体的任务调整其表示方式

# 1.3 词嵌入表示(Word Embedding)

  • 使用一个连续、地位、稠密的向量来表示词，成为==词向量==
  • 与分布式表示不同的是其赋值方式，词向量中的向量值随着目标任务的优化过程自动调整，即可以将词向量中的向量值看作模型的参数
  • 如果目标任务的训练数据比较少，通常利用自然语言文本中所蕴含的自监督学习信号(词与上下文的共现信息) 先来==预训练==词向量

# 1.4 文本的词袋(Bag-of-Words,BOW)表示

  • 假设文本中的词语是没有顺序的集合，将文本中的全部词所对应的向量表示（既可以是独热表示，也可以是分布式表示或词向量）相加，即构成了文本的向量表示
  • 在使用独热表示时，文本向量表示的每一维恰好是相应的词在文本中出现的次数
  • 优点：简单，直观

缺点：

  1. BOW相同的词可能其顺序不同
  2. 无法融入上下文信息，可以通过引入**二元词(Bigram)**词表解决，但随词表的增大，会引入更严重的Data Sparsity问题

# N元语言模型

  • 根据已有词序列对下一时刻可能出现的词的条件概率进行估计

例如通过历史“我喜欢”预测下一个词为“读书”的概率 在给定一个语料库时，该条件概率可以理解为当语料中出现“我喜欢”时，有多少次下一个词为“读书”，然后通过最大似然估计进行计算

其中C表示相应词序在语料库中出现的次数

  • 通过以上的条件概率，可以用相应的单词的联合概率P(w1, w2, ..., wl)计算一个句子出现的概率

  • 马尔科夫假设：“下一个词出现的概率只依赖于它前面n-1个词”

  • N元语法(N-gram)模型：满足马尔科夫假设的模型
